Data types in C++ vary in many ways. the amount of bytes, or the size, of an object can vary depending on what the object is. When it comes to pointers, pointers dont vary in size depending on what they are pointing to, but rather the memory location that they are pointing to; which changes depending on the system running it. Arrays vary in size depending on what they are assigned since an array has to have a size assigned to it, where as vectors dont. Vetors always show the constant size.

Questions

Do all pointer types (T*) need the same storage?

The pointer types don't need to have the same storage since they are all assigned to different locations in memory.

Does the size of a pointer (T*) reflect the size of thing pointed at(T)?

No, the size of a pointer doesn't reflect the thing it is pointing to, rather it's size is reflecting to the memory location it is pointing to.

Does the size of an array vary with the number of elements?

The size of the array will change on the number of element that it contains; or that its assigned to it. Arrays require a value given to them.

Does the size of a vector vary with the number of elements? (and where does the rest of the vector get hidden?)

A vector will vary with the number of elements within it, but it does it dynamicly. The value of the vector doesnt change itself since it manages memory personally. When the value is checked it alwasy brings the same size because when its being checked for its value its only showing the value of its scope and not of the array(s) within the vector. the rest of the vecotor is not hidden, again its just showing its value at its scope before the array within is even accessed.
